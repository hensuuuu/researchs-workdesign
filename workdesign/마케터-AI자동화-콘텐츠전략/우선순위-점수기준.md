## 기준 2. 바이럴 포맷 적합도 — 20점

각 항목 충족 시 4점, 5개 항목 합산:

| 체크 항목 | 판단 기준 |
|---------|---------|
| (a) 구체적 숫자 포함 | 제목/썸네일에 숫자 1개 이상 |
| (b) Before/After 또는 비교 구조 | 전후 비교 또는 A vs B 구조 |
| (c) 따라할 수 있는 워크플로우 | 단계별 실행 가이드 포함 |
| (d) 실제 결과/성과 명시 | 검증된 수치 또는 직접 경험 |
| (e) 시간/비용 절약 명시 | 절약량 또는 효율 수치 포함 |

**근거:** 숫자+결과형 제목 유튜브 조회수 x4.8배, LinkedIn 캐러셀 +596%, 인스타 완주율 90%+ 릴스 알고리즘 부스트

## 기준 6. 제작 난이도 (더미 대체 가능성) — 12점

실제 데이터(광고 계정, GA4, CRM 등) 없이 더미 데이터로 콘텐츠 시연이 가능한 정도.

| 조건 | 점수 |
|------|------|
| 더미 데이터를 AI가 즉시 생성 가능 (프롬프트만으로 시연 완결) | 12점 |
| 더미 데이터 필요하지만 직접 만들기 쉬움 (간단한 CSV, 텍스트) | 9점 |
| 더미 가능하지만 구조가 복잡해 준비 시간 필요 | 6점 |
| 실계정 필요하지만 무료 테스트 계정으로 대체 가능 | 3점 |
| 실계정·실데이터 없으면 시연 불가 (GA4 연동, 광고 계정 등) | 1점 |

---

## 기준 7. MVP 구현 난이도 — 10점

실제 데모 제품 구현 난이도. 쉬울수록 영상 생산속도가 올라가므로 높은 점수.

| 조건 | 점수 |
|------|------|
| 프롬프트만으로 즉시 데모 가능 (코딩 없음) | 10점 |
| 노코드 툴(n8n 등) 조합으로 1~2시간 내 구현 | 8점 |
| 간단한 스크립트 필요 (반나절 이내) | 5점 |
| API 연동 + 환경 세팅 필요 (1~2일) | 3점 |
| 복잡한 아키텍처 or 외부 계정 다수 필요 | 1점 |
